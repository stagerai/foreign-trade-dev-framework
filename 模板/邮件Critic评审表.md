# 邮件Critic评审表（独立审核视角）

**作用:** Writer起草后，Critic独立审核，发现问题
**触发条件:** 每次邮件起草完成后强制执行 — 无例外

---

## Critic铁律（最高优先级）

| 角色 | 职责 | 禁止事项 |
|------|------|---------|
| **Writer** | 起草邮件、自我评分、提交草稿 | 禁止修改Critic评分 |
| **Critic** | 独立审核、盲评打分、发现问题 | 禁止修改邮件内容 |

**强制执行:** 每次邮件起草必须经过Critic评审，无例外。
**严格分离:** Writer写完就不能改评分，Critic评完就不能改内容。

---

## 步骤3：Writer评分 vs Critic评分对比

| 维度 | Writer评分 | Critic评分 | 差距 |
|------|-----------|-----------|------|
| 1. 主题行具体性 | /20 | /20 | ± |
| 2. 主题行长度 | /10 | /10 | ± |

| 4. 差异化 | /20 | /20 | ± |

| 7. 联系人适配 | /10 | /10 | ± |
| **总分** | /100 | /100 | ± |

**差距判定：**
| 差距情况 | 处理 |
|---------|------|
| Critic评分≥Writer评分 | Writer评分可信，取Writer评分 |
| Critic评分<Writer评分，差距≤5分 | 均可接受，取Critic评分（较低值） |
| Critic评分<Writer评分，差距>5分 | Writer可能存在盲点，需针对性修复 |

## 步骤4：Critic盲点检测

| 盲点类型 | 是否触发 |
|---------|---------|
| **自嗨主题** — "主题很有创意"但看了不知道说什么 | □ 是 □ 否 |
| **过度自信案例** — 案例和客户业务无关 | □ 是 □ 否 |
| **CTA新鲜度误判** — 客户看到太多Interested了 | □ 是 □ 否 |
| **篇幅自我宽容** — 超出10%以上，客户懒得看 | □ 是 □ 否 |
| **差异化幻觉** — 实际和历史角度重复 | □ 是 □ 否 |

## 步骤5：Critic最终判定

### 维度最低门槛检查（Critic视角）

| 维度 | Critic得分 | 最低门槛 | 是否达标 |
|------|-----------|---------|---------|
| 主题行具体性 | /20 | ≥10分 | □达标 □不达标 |
| 主题行长度 | /10 | ≥8分 | □达标 □不达标 |
| 差异化 | /20 | ≥12分 | □达标 □不达标 |
| 价值具体性 | /15 | ≥10分 | □达标 □不达标 |
| CTA门槛 | /10 | ≥8分 | □达标 □不达标 |

### Critic最终判定

| 条件 | 判定 |
|------|------|
| Critic评分≥95 + 所有维度≥门槛 + 无盲点触发 | 通过，可提交 |
| Critic评分≥95 + 某维度<门槛 | 需针对性修复 |
| Critic评分80-94 + 无盲点触发 | 需优化，进入修复循环 |
| Critic评分<80 或 盲点触发 | 需重写或大改 |

**Critic最终判定：**
□ 通过，可提交给Owner审核
□ 需针对性修复（修复后Critic再评）
□ 需重写（返回Writer重新起草）

---

## Critic评审后流程

1. **Critic判定通过:** 邮件提交给Owner审核，等待发送
2. **Critic判定需修复:** Writer根据Critic建议修复，修复后重新提交Critic评审
3. **Critic判定需重写:** Writer重新起草，新邮件重新进入Writer-Critic流程

**每次修复后必须再次Critic评审，直到通过。**
