# 多客户批量开发流程（长期运行保障）

**重要性：** 核心运营流程
**适用范围：** 多个客户（20+）长期批量开发
**核心目标：** 系统化管理全部客户的开发进度，保障质量

---

## 核心问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| **质量失控** | 多客户时容易偷懒、跳流程 | 强制流程检查 + 评分机制 |
| **进度混乱** | 哪个客户到哪轮？需检查哪个？ | 开发状态总览文件 + 检查日期追踪 |
| **会话断掉** | 会话重启后忘记进度 | 状态总览文件作为唯一来源 |
| **节奏失控** | 一天写太多邮件质量下降 | 每日邮件上限 + 轮次间隔 |
| **遗漏客户** | 某客户长期未跟进 | 检查日期自动提醒 |

---

## 核心机制：开发状态总览

> "开发状态总览是唯一来源。会话重启后，读取此文件即可恢复全部进度。"

**文件位置：** `开发状态总览.md`（项目根目录）

---

## 第一阶段：客户优先级排序

### 会话启动时强制读取

```
1. 读取 CLAUDE.md
2. 读取 开发状态总览.md — 关键
3. 读取 memory/YYYY-MM-DD.md（今天）
```

### 确定今日开发对象

| 状态 | 筛选条件 | 操作 |
|------|---------|------|
| **已到期** | 检查日期 ≤ 今天 | **今日必须处理** |
| **检查** | 检查日期 = 今天 | 今日检查回复 |
| **待发送** | 已起草，等待发送 | 提醒 Owner 发送 |
| **有回复** | 有回复待处理 | 今日处理回复 |
| **正常** | 检查日期 > 今天 | 等待，不处理 |

**今日开发优先级：**
1. 已到期客户（最高优先）
2. 有回复客户（高优先）
3. 检查客户（中优先）
4. 待发送客户（提醒 Owner）

---

## 第二阶段：批量起草流程

### 逐客户执行（不并行）

> "一个客户完整执行完流程，再开始下一个。不并行，避免混乱。"

**单客户完整流程：**
```
1. 读取客户档案
2. 读取开发日志
3. 读取联系人记录
4. 输出起草前现状报告
5. 确定本轮策略
6. 撰写邮件（多联系人差异化）
7. 评分检查（每封邮件单独评分）
8. 存储到开发日志
9. 输出邮件完成报告
10. 标记状态：待发送
```

---

## 第三阶段：质量保障机制

### 每封邮件单独评分

> "多客户开发时，最容易偷懒跳流程。必须每封邮件单独评分。"

**提交前必须确认：**
```
□ 所有邮件已单独评分
□ 所有邮件得分 ≥70分
□ 所有邮件已存储到开发日志
□ 所有邮件状态标记 待发送
□ 所有邮件主题行与历史不重复
□ 所有邮件正文与历史不重复
□ 所有邮件CTA低门槛
□ 已更新开发状态总览
```

---

## 第四阶段：归档与更新

### Owner 通知"已发送"后

| 步骤 | 操作 | 文件 |
|------|------|------|
| **1** | 开发日志缩为摘要 | `各客户/01_开发日志.md` |
| **2** | 完整原文写入各联系人记录 | `各客户/联系人沟通记录/[姓名].md` |
| **3** | 更新开发状态总览 | `开发状态总览.md` — 关键 |
| **4** | 计算下次检查日期 | 发送日期 + 间隔天数 |
| **5** | 回复 Owner 确认 | 会话中 |

---

## 第五阶段：检查日期追踪

### 检查日期计算

| 轮次 | 间隔天数 | 检查日期计算 |
|------|---------|-------------|
| 第1轮 | 7天 | 发送日期 + 7 |
| 第2轮 | 7天 | 发送日期 + 7 |
| 第3轮 | 7天 | 发送日期 + 7 |
| 第4轮 | 14天 | 发送日期 + 14 |
| 第5轮+ | 14天 | 发送日期 + 14 |

### 检查回复流程

```
1. 打开邮箱客户端
2. 搜索客户域名（如：example.com）
3. 查看是否有新回复
4. 如有回复 → 更新状态总览为有回复 → 处理回复
5. 如无回复 → 更新检查日期 → 继续等待
```

---

## 第六阶段：会话重启后继续

### 重启后强制读取

```
1. CLAUDE.md — 规则与索引
2. 开发状态总览.md — 关键：恢复进度
3. memory/YYYY-MM-DD.md（今天）— 当日记录
4. memory/SESSION_REPORT.md — 上次会话报告
```

### 从状态总览恢复进度

1. 查看"今日待办"部分
2. 查看"已到期"客户
3. 查看"有回复"客户
4. 确定今日开发对象
5. 继续执行开发流程

### 每周统计（周日）

| 项目 | 本周数据 | 上周数据 | 变化 |
|------|---------|---------|------|
| 起草邮件数 | | | |
| 发送邮件数 | | | |
| 收到回复数 | | | |
| 回复率 | | | |
| 新增客户数 | | | |
| 成交客户数 | | | |

### 每周复盘

1. 哪个客户回复了？为什么？
2. 哪个客户长期沉默？是否需要调整策略？
3. 本周邮件评分平均多少？质量是否下降？
4. 有遗漏客户吗？检查日期是否过期？
