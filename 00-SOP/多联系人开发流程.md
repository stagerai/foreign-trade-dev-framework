# 多联系人同时开发流程

**重要性：** 高频场景，必须掌握
**适用范围：** 一个客户有多个联系人时的开发策略
**前提：** 已阅读 `邮件起草完整流程.md`

---

## 核心原则

### 毅冰原则

> "多联系人不是群发，是精准打击。不同角色，不同角度，不同时间。"

**核心原则：**
1. **不群发相同内容** - 每个联系人收到不同的邮件
2. **按优先级分批发送** - 先发P0/P1，观察反应后再发P2/P3
3. **按角色定制角度** - 决策者看ROI，采购看价格，技术看规格
4. **有人回复后暂停其他** - 避免信息混乱，先处理回复

---

## 第一阶段：联系人分级（强制执行）

### 按角色分类

**角色分类表：**

| 角色 | 判断标准 | 邮件角度 | 典型职位 |
|------|---------|---------|---------|
| **决策者** | CEO/总经理/Gerente | ROI + 战略价值 | 总经理、高管 |
| **采购** | 采购/Compras/Procurement | 价格 + 交期 + 服务 | 采购负责人 |
| **技术** | 技术/运营/Soporte/IT | 规格 + 案例 + 可靠性 | 项目总监、技术支持 |
| **市场** | 市场/Marketing/商务 | 产品线扩展 + 利润空间 | 市场经理、销售负责人 |
| **行政** | HR/财务/Admin | 请求转交 + 信息传递 | HR、财务 |

### 确定本轮发送对象

**发送优先级矩阵：**

| 轮次 | 发送对象 | 原因 |
|------|---------|------|
| **第1轮** | 仅P0/P1 | 先攻核心，观察反应 |
| **第2轮** | 仅之前P0/P1 | 继续跟进核心 |
| **第3轮** | 仅之前P0/P1 | 破冰/供应商角度 |
| **第4轮** | 仅之前P0/P1 | 破冰询问 |
| **第5轮（无回复）** | 新增P2/P3 | 多路线开发，扩大覆盖 |
| **第5轮（有回复）** | 仅回复者 | 处理回复，暂停其他 |

> "前4轮只攻P0/P1。如果4轮后仍无回复，第5轮才扩展到P2/P3。"

---

## 第二阶段：邮件差异化设计（强制执行）

### 不同角色不同角度

| 角色 | 邮件角度 | 正文核心 |
|------|---------|---------|
| **决策者** | ROI + 战略价值 | 降低成本X%，供应商多元化 |
| **采购** | 价格 + 交期 + 服务 | X%价格优势，X周交期，X年质保 |
| **技术** | 规格 + 案例 + 可靠性 | 核心参数 + 案例证明 |
| **市场** | 产品线扩展 + 利润空间 | 扩展产品线，利润空间X% |
| **销售** | 客户案例 + 销售工具 | 成功案例，销售工具支持 |
| **行政** | 请求转交 + 信息传递 | 请转交采购部门，简单礼貌 |

### 主题行差异化（绝不重复）

> "同一轮不同联系人，主题行必须不同。重复=客户发现群发=删除。"

| 差异维度 | 示例（同一轮5人） |
|---------|------------------|
| **角色角度** | ROI for [公司] / Pricing for [公司] / Specs for [公司] |
| **具体数字** | 30% lower / 40% fewer callbacks / 15ms recovery |
| **案例/痛点** | Remote projects / Urgent orders / Harsh environments |
| **称呼** | Hi [Name1] / Hi [Name2] / Hi [Department] |

### 正文差异化（绝不重复）

| 角色 | 具体元素 |
|------|---------|
| **决策者** | 降低成本$XX/年，利润提升XX% |
| **采购** | 比竞品A便宜X%，比竞品B更好服务 |
| **技术** | 核心参数 + 案例城市/行业 |
| **市场** | 扩展产品线，X%利润空间，批发价格 |
| **销售** | 成功案例，销售工具，客户背书 |
| **行政** | 请转交采购部门，简单礼貌 |

---

## 第三阶段：发送节奏控制

| 轮次 | 发送时间 | 发送对象 |
|------|---------|---------|
| **第1轮** | 同一天发送 | P0/P1 |
| **第2轮** | 7天后 | 同上P0/P1 |
| **第3轮** | 7天后 | 同上P0/P1 |
| **第4轮** | 7天后 | 同上P0/P1 |
| **第5轮（无回复）** | 14天后 | P2/P3 |
| **第5轮（有回复）** | 24小时内 | 仅回复者 |

---

## 第四阶段：回复协调处理

### 有人回复后处理流程

| 步骤 | 操作 |
|------|------|
| **1** | 停止发送其他联系人 |
| **2** | 24小时内回复 |
| **3** | 针对回复内容回应 |
| **4** | 更新开发日志 |
| **5** | 更新联系人记录 |
| **6** | 决定下一步策略 |

### 回复类型处理

| 回复类型 | 处理策略 | 其他联系人处理 |
|---------|---------|--------------|
| **积极回复（索取资料）** | 立即发送资料，24小时内 | 暂停发送，专注处理 |
| **询问价格/规格** | 发送报价/规格表 | 暂停发送 |
| **"暂不需要"** | 礼貌感谢，转长线培养 | 可继续开发其他联系人 |
| **"转交给采购"** | 联系采购部门 | 暂停，等待采购回复 |

### 多人回复处理

| 回复来源 | 优先级 | 处理顺序 |
|---------|--------|---------|
| **决策者回复** | 最高 | 第一处理 |
| **采购回复** | 高 | 第二处理 |
| **技术回复** | 中 | 第三处理 |
| **销售/市场回复** | 中低 | 第四处理 |
| **行政回复（转交）** | 低 | 最后处理 |
